# Design Note

Executive search is not a summarisation problem. It is a decision-support problem under uncertainty. In PPP's world, the real task is not to produce five neat profile write-ups. It is to help a consultant decide, from uneven public evidence, who deserves the first call, who belongs in a second-wave screen, and who should stay in the market map. Public signals are patchy, identity confidence is uneven, and remit shape is often only partially visible. That means the system has to support call-priority judgment without pretending the evidence is cleaner than it is.

The system plugs directly into the existing consultant workflow: mandate intake, candidate review, call-priority decision, then live screening. It does not introduce a new process; it compresses the one already in use. The role specification anchors what the mandate needs, and each candidate is normalized into a structured evidence profile before the final brief is written. The middle step in the workflow is not “write a summary,” but “decide who to call and what to test.” The output is therefore a recruiter decision artifact, not a résumé synopsis.

A key design choice is that the system explicitly structures relative priority. The ranking and score are designed to express phone-call order, not just a generic label such as “good.” In practice, the output answers an operational question: who is the most commercially sensible first call, who follows, and who is credible but lower urgency.

The operational value is time compression. First, it reduces first-pass research time by converting raw inputs into a consistent recruiter-ready shape. Second, it reduces comparison effort by enforcing the same role-fit dimensions across candidates. Third, it lowers the cognitive load of uncertainty handling by carrying evidence distinctions directly into the output. This shifts the consultant’s time from information gathering to higher-value judgment and conversation.

Auditability is explicit. The structured JSON output is the review artifact. Evidence normalization produces recruiter-facing signals and confidence states that can be inspected rather than hidden in prose. This makes the reasoning behind each ranking inspectable. A QA layer runs before export so the final bundle is valid and internally consistent. Fixture-backed reproducibility provides a stable review path even when public evidence changes.

That evidence discipline shapes the trade-offs. Stronger public support allows a more assertive shortlist-style note, while weaker identity confidence remains cautious. Fixture mode stabilizes the evidence pack for review, while live mode demonstrates the same workflow on changing public sources.

With more time, I would not add more features first. I would first improve evidence fidelity around channel ownership, team scale, and confirmed remit scope, because those are the details that most directly change recruiter judgment. Second, I would calibrate the scoring and language against consultant review so the output more clearly separates priority-call names from adjacent but lower-conviction profiles. Third, I would make the reviewer experience more explicit by labeling profiles as shortlist, screen next, map-only, or verify first, so the output behaves even more like an operational search tool rather than a polished export.

One additional automation I would build for PPP is a mandate-to-market-map calibration workflow. After a consultant inputs a role brief, the system would review a wider slate and automatically sort names into shortlist, screen, map-only, and off-lane buckets, while also generating the one question that most needs live validation for each person. The value is not just faster research; it is faster market calibration. That would help consultants move from intake to first-call plan more quickly, standardize early-stage search judgment across projects, and create a reusable internal research asset rather than a one-off output file.

A consultant would use this because it does not change their workflow, only compresses it. It improves consistency without removing judgment. The system makes decisions faster, more comparable across candidates, and more disciplined about what the public record supports.
