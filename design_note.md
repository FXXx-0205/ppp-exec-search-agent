# Design Note

I chose a two-stage architecture because PPP’s task is not just “generate some text from a CSV”; it is closer to a recruiter workflow where evidence quality matters as much as output shape. In a single prompt, research, reasoning, and formatting tend to get mixed together, which makes hallucination harder to control. In this submission, the first stage produces a structured enrichment package for each candidate: verified public snippets, inferred tenure clues, channel indicators, firm context, missing fields, uncertain fields, and confidence notes. The second stage uses Claude to turn that bounded evidence package into the final candidate briefing JSON. I also enforced strict schema validation and a post-generation QA layer because recruiter-facing outputs should fail loudly when they are malformed, generic, or overconfident, rather than quietly passing through as “close enough.” That is why the system validates not only structure, but also sentence count, evidence linkage, uncertainty language, and obvious placeholder text.

With more time, I would replace the fixture-backed enrichment layer with a real public research connector that searches company pages, leadership bios, regulatory disclosures, and other public web sources relevant to Australian funds management. I would also add a source citation layer so a consultant could see which claim in the briefing came from which source. A third improvement would be a recruiter feedback loop: if a consultant says a mobility assessment was weak or a fit justification overstated the case, that feedback should be captured and used to improve later runs. Finally, I would calibrate confidence and scoring more carefully. In this version, the system is intentionally conservative; with more time, I would make the confidence model more nuanced by separating “publicly verified,” “strongly inferred,” and “insufficient evidence” in a more explicit way.

If I joined PPP, one additional automation I would build is a “search mandate to longlist triage assistant.” In practice, a search does not stop at candidate briefing; the harder operational problem is deciding who deserves immediate partner attention, who should stay in the broader map, and who is adjacent but not ready. I would build a workflow that takes a live mandate, applies role-specific criteria, scores candidate evidence quality, flags missing proof points, and produces a prioritised outreach queue for consultants. For PPP’s funds management distribution work, that would be especially useful because channel relevance, institutional versus wholesale depth, leadership scope, and market credibility are often unevenly visible in public profiles. A triage assistant would help consultants move faster without giving up judgment, and it would fit naturally with the candidate briefing system delivered here.
